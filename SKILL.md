# drug-intel — 药物研发竞争情报（美国 + 中国）

这是一套**运行时流程**。当有人说「查一下 X 药」时，你（Claude）按本文件从头到尾执行。
本工具只看 **美国 + 中国** 两个地区，其它地区不展开。

> ⚠️ 免责：本工具产出**仅供研发情报参考，非医疗建议、非投资建议**。所有高幻觉字段必须带徽标并经人工复核。

> **本工具的价值一句话**：不是给一个漂亮答案，而是**诚实标注每个字段的可信度，把不确定的东西清清楚楚交到人手上核对**。
> 宁可 `❓N/A` 也不编。

## 1. 最高优先级规则：全局来源标注（badges）

**报告里每一个字段都必须带来源徽标。** 「所有高幻觉区」和「判定为找不到的区域」一律**显式标注**，
绝不静默省略、绝不无来源直接给结论。

| 徽标 | 含义 | 用法 |
|------|------|------|
| 🏛️官方 | **官方源·最高可信（仅限客观事实）** | 来源是 **(a) 该药研发/上市企业官网**（如 astrazeneca.com、daiichisankyo.com、astrazeneca.com.cn）**或 (b) 政府/监管官网**（fda.gov、accessdata.fda.gov、purplebooksearch.fda.gov、nmpa.gov.cn、cde.org.cn、ema.europa.eu、clinicaltrials.gov）。**只对客观事实生效**（见 1.2）。必须附 URL |
| ✅`<数据库名>` | 权威数据库可核实 | 例：✅PubChem、✅ChEMBL、✅ClinicalTrials.gov、✅openFDA。机器可查到。与 🏛️官方 同属**最高一档** |
| 🔍web | 联网检索到的（非官方/二手） | **必须附 URL**。例：🔍web([targetedonc](#))。权威医药媒体属此档 |
| ⚠️ | 来源存疑 / 单一弱来源 | 建议人工确认；常见于"机构新闻稿""二手聚合站"级来源 |
| 🔴 | 高幻觉区 或 必经人工确认 | **绝不允许无来源直接给结论**。即使查到也要标，并说明不确定性来源 |
| ❓N/A — 未找到 | 检索过但确实没找到 | **必须写明查过哪些源**。这本身也要显式标出，不能省略 |

**可信度排序（事实型字段，由高到低）：** `🏛️官方 ＝ ✅库 > 🔍web > ⚠️ > ❓N/A`。
**来源优先级（一律遵守）：** 公司官网 / FDA / CDE·NMPA 官网（→🏛️官方）＞ 权威医药媒体·数据库（→✅ 或 🔍web）＞ 其他。
查不到就写 `❓N/A` 并注明查过哪些源。**绝不编造事实、数字或来源。**

### 1.2 🏛️官方徽标的边界（务必照此，别用过头）

🏛️官方 = "这条**客观事实**来自最权威的源，最可信"。它**只能**贴在**客观事实型字段**上：

> 是否上市 ｜ 批准日期 ｜ 获批适应症 ｜ 作用机制(MoA) ｜ 批件/独占状态 ｜ 专利号 ｜ 试验编号(NCT/CTR)。

**以下字段即使来自官网，也必须仍保留 🔴**（官方源能提升"事实"的可信度，但**不能解除"主观/宣传字段"的强制 🔴**）：

- **销售额 / 任何财务数字** —— 官网/财报口径仍易误读、易过期、易混淆地区与币种。写法：`🔴（来源🏛️官方财报，但财务口径仍须人工复核）`。
- **"同类最佳 / 核心优势"等主观对比** —— 官网本质是宣传，天然有立场。写法：`🔴（含🏛️官方原文引用，但属企业宣传立场，须独立复核）`。

也就是说：一个字段可以**同时**带"来源徽标"和"🔴"。`🏛️` 回答"信源多硬"，`🔴` 回答"这字段本身要不要人工兜底"——两者不冲突，遇到上面两类字段就**两个都标**。

### 1.3 强制 🔴 的字段（不管查没查到、不管来源多权威，都标 🔴 并解释不确定性来源）

1. 上市所依据的**关键(pivotal)试验 NCT/CTR 号** —— CT.gov 无 "pivotal" 字段，需从 FDA 说明书
   *Clinical Studies* 段 / CDE 上市依据反查。（注：试验**编号本身**若来自 FDA/官网可叠加 🏛️，但"它是 pivotal"这一判断仍 🔴）
2. **任何药物销售额数字** —— 只在公司财报/年报，非产品页，且只有大品种单独披露。（见 1.2：官网也不解除 🔴）
3. **第 9 步核心优势**（AI 主观对比）—— 最主观高幻觉区。（见 1.2：引用官网也不解除 🔴）
4. **LOE 日期（中美）** —— 无现成 "LOE=某日" 字段，需综合专利+独占期推算。
5. **中国侧的任何审批/上市/仿制信息** —— 无干净 API，天然低可信。

## 2. 暂停门 (Pause Gates) — 必经人工确认后再继续

> 「暂停门」是内部术语，**给用户看到的话里绝不出现这个词**（见 2.1 的重写话术）。

为防止错误向下游传播，**G0–G5 全部为默认硬停**：到达时**停下来、把该门的中间结论给用户、等确认后再继续**
（不是仅打徽标）。**唯一例外**：用户开场明确说「全部一次跑完别停」时，整批降级为"高亮 🔴 + 写进核对清单"（见 2.2）。

| 门（内部编号） | 位置 | 内部触发逻辑（不直接说给用户） |
|----|------|------|
| 🚧 **G0 身份门** | 开场 A（分子消歧） | **硬停**：实体错 = 整篇报告全错。多候选必须让用户选定再跑 |
| 🚧 **G1 口径门** | 开场 B（阶段口径） | **硬停**：决定全文结构。先确认再继续 |
| 🚧 **G2 pivotal 门** | 第 4 步（上市依据 NCT/CTR） | **硬停**：喂给 LOE、说明书声明；映射极脏。给候选 + 来源，等用户拍板再继续 |
| 🚧 **G3 LOE 门** | 第 6 步（中美 LOE） | **硬停**：业务影响大、需推算。给"日期+依据+来源"，等用户确认推算逻辑再继续 |
| 🚧 **G4 销售额门** | 第 5 步（销售数字） | **硬停**：高幻觉高影响。给"数字+财报出处"，未取到则 ❓N/A，等用户确认再继续 |
| 🚧 **G5 中国门** | 第 4/6/7 步中国侧任何审批/上市/仿制结论 | **硬停**：无干净源。每条都等用户到官方站核对后再定稿 |

> 第 9 步核心优势虽是 🔴，但它是最后一步且已显式标注"AI 主观、跨试验比较不严谨"，
> 默认**重度打标 + 进核对清单即可，不设硬停**（除非用户要求逐条确认）。

### 2.1 各门"给用户看到的那句话"——必须用大白话（照下面说，可更自然）

**铁律**：内部逻辑、门编号、流程结构**全部不变**；只把"对用户说出口的话"换成正常人说话。
**禁止**对用户说出这些词：暂停门 / 口径 / 口径门 / pivotal / pivotal 门 / MAH / LOE / 消歧。
必须用到时就地翻译成人话（如 "MAH"→"在中国负责上市、卖这药的公司"；"LOE"→"专利大概什么时候到期"）。

- **G0 身份确认**（替代"分子身份消歧"）：
  > 「我先跟你确认一下：你要查的是不是 **Enhertu（中文名优赫得，也叫 trastuzumab deruxtecan / 德曲妥珠单抗 / T-DXd / DS-8201）** 这个药？是的话我就开始查；如果你指的是别的，告诉我一声。」
  > （若有多个候选：「你给的名字能对到下面几个，麻烦选一个：1)… 2)… 3)…」）

- **G1 阶段口径**（替代"全局最远 vs 按适应症口径"）：
  > 「查之前定个方向：你是想知道**这个药目前最领先、最快做到了哪一步**（简短，给你最高进度一句话），还是想看它**每一种适应症（比如乳腺癌、胃癌、肺癌）分别做到哪了**（更长更细）？」

- **G2 关键临床试验**（替代"pivotal 门"）：
  > 「这药能上市，是靠几项关键临床试验撑起来的。不过这块网上信息很容易对不上号，所以我会先把我找到的'最可能是关键试验'的编号列给你，但**最终建议你拿官方说明书再核一遍**——我会把它们都标红提醒你。」

- **G3 专利到期**（替代"LOE 门"）：
  > 「关于**专利大概什么时候到期**（到期后别家才可能做仿制／生物类似药）——这个没有一个现成的'到期日'，我只能根据专利和保护期帮你推算。这块我没法 100% 打包票，会把推算依据列出来、并标红让你最后自己核。」

- **G4 销售额**（替代"销售额门"）：
  > 「关于**这药卖了多少钱**——销售额只在公司财报里有，而且口径容易看错、容易过期。我会给数字配上财报出处，但这块也会标红，请以公司最新年报／季报为准；查不到我就如实写'没找到'，绝不会编一个数字给你。」

- **G5 中国侧**（替代"中国门"）：
  > 「**中国这边的审批、上市、有没有仿制药**——中国没有一个像美国那样干净的公开数据库，信息得靠官网一条条查，最容易出错。所以中国相关的每一条我都会标红，建议你最后到 NMPA（国家药监局）官网再核一遍。」

> 多个门可在同一处一并提请确认（如 G0+G1 开场一起问，用 `AskUserQuestion` 最干净），但未得确认不得跨过该门继续。

### 2.2 "一次跑完"模式 —— 连这个选择本身也用大白话问

到场默认走逐门确认。但要**主动用大白话**把"要不要一次跑完"这个选择交给用户，比如：

> 「还有一件事：报告中间有几个地方信息特别容易出错、最好你来拍板。你想要我**每遇到一个不太确定的地方就停下来问你一句**，还是**一口气先把完整初稿写出来、把所有需要你再核对的地方都用红色集中标在最后**？（不确定的话建议选后者，省事，最后照红色清单核就行。）」

- 选"逐个停" → 默认逐门硬停。
- 选"一次出" / 用户开场就说「别停直接出」→ 把 **G2–G5 降级为"高亮 🔴 + 写进核对清单"**（G0/G1 仍至少复述确认一次），
  并在报告顶部显著声明："⚠️ 已按你的要求一次跑完、没有逐项停下确认，下面所有标红的地方都还没经过你核对。"

## 3. 中国侧专用来源（顺序）

中国临床/审批信息优先用官方源，按序：

1. **药物临床试验登记与信息公示平台** —— https://www.chinadrugtrials.org.cn/index.html （CDE 官方，**首选**）
2. **NMPA / CDE 官网** —— 审批、上市、一致性评价：https://www.nmpa.gov.cn/ ， https://www.cde.org.cn/
3. **权威医药媒体**（如药智、insight / 医药魔方、摩熵医药等报道）

> ⚠️ `chinadrugtrials.org.cn` **无公开 API、且为动态加载**，web 抓取可能不稳定。
> 取到且来自上面 1/2 的官方站 → 可标 **🏛️官方**（仅对客观事实，如批准日期/批件状态）；
> 取自第 3 类媒体 → 🔍web(URL)；取不到 → 标 `❓N/A` 并**提示用户手动到该站检索**（给出站点链接）。
> **中国侧整体默认高人工介入**，所有中国侧结论默认走 🔴 / G5 暂停门（即使叠加了 🏛️官方，🔴 也不解除）。

## 4. 开场确认（只问一次，再开跑）

收到「查 X 药」后，**先把脚本跑起来**（见第 5 节）拿到 `identity_rollup`，然后**一次性**用 2.1 的大白话问清：

**A. 它是哪只药？（🚧 G0 — 对用户说 2.1 的 G0 话术）**
用户输入可能是分子名 / 中文名 / 实验代号 / 商品名。解析出唯一实体：INN、PubChem CID、ChEMBL ID、别名、盐型/游离碱。
- 用脚本的 `identity_rollup` + `pubchem.synonyms` 给出候选。
- **多候选 → 列出让用户选；单一明确候选也复述一遍请确认。**

**B. 要看最高进度还是逐个适应症？（🚧 G1 — 对用户说 2.1 的 G1 话术）**

**C. 要不要一次跑完？（用 2.2 的大白话问；可与 A/B 合并到一次 `AskUserQuestion`）**

> 三件事都确认后才进入九步工作流。

## 5. 数据脚本（确定性源，永远先跑它）

```bash
python scripts/fetch_drug_data.py "<药名或代号>"
# 可选： --max-studies 250   --no-studies-list   --compact
```

- 纯 stdlib（urllib），无需 pip。返回合并 JSON 到 stdout。
- **这是工作流的第一步，绝不绕过**（即使准备出可视化报告，也先跑脚本拿确定性数据）。
- 覆盖**有干净 API 的确定性源**，这些字段可直接打 ✅：

| JSON 路径 | 来源 | 喂给哪步 | 徽标 |
|-----------|------|---------|------|
| `deterministic_sources.pubchem` | PubChem | 步1 身份/结构/式/SMILES/PNG | ✅PubChem |
| `deterministic_sources.chembl.molecules / best_match` | ChEMBL | 步1 ChEMBL ID、max_phase | ✅ChEMBL |
| `deterministic_sources.chembl.mechanisms` | ChEMBL | 步3 MoA（数据库级） | ✅ChEMBL |
| `deterministic_sources.clinicaltrials_gov.*` | CT.gov v2 | 步4 阶段/时间线/计数 | ✅ClinicalTrials.gov |
| `deterministic_sources.openfda.drugsfda` | openFDA | 步2 厂家(US)、步5 批准年、步7 ANDA信号 | ✅openFDA |
| `deterministic_sources.openfda.ndc` | openFDA | 步7 在售/仿制类别 | ✅openFDA |

- **关键约定**：任何源失败/查无 → 该源在 JSON 里是 `null`，`source_status` 写明原因。
  **`null` 永远不等于"药不存在"**。早期代号在 openFDA 常查不到属正常；**生物药/ADC 在 PubChem 常无小分子记录，应标 ❓N/A（已查 PubChem 化合物库），属正常**。
- `web_and_manual.*` 字段脚本一律返回 `null` + 该去哪个官方站的 URL —— 这些走 web + 人工。

> **网络说明**：脚本要访问 pubchem / ebi / clinicaltrials.gov / api.fda.gov 等域名。
> 如果运行环境对联网域名有白名单限制、脚本所有源都报 `error`，**这是环境设置问题、不是药不存在**——
> 见 `GETTING_STARTED.md` 的"域名放开"一节（把允许域名设为 `*` 或补上清单即可）。
> CT.gov 的 `us.highest_phase` 是**临床试验阶段**，**不等于已上市**。是否上市看 openFDA（美国）/ NMPA（中国）。

## 6. 工作流九步（+ LOE + 仿制）

> 每步产出都要带徽标。能用脚本 ✅ 的优先 ✅；来自官网/监管站的**客观事实**叠加 🏛️官方；脚本/官方没有的去 web（🔍 附 URL）；
> 中国侧 / LOE / 销售额 / pivotal 走 🔴 + 暂停门（**🔴 不因来源是官方而解除**，见 1.2）。

### 步骤 1 · 分子名确认
- 中英文互校。脚本已在 PubChem / ChEMBL 命中；另在 **openFDA、CDE/NMPA、ClinicalTrials.gov、chinadrugtrials.org.cn** 检索该名称/代号并抓取命中记录。
- 输出：INN、中文名、CID、ChEMBL ID、盐型/游离碱、别名、分子式、结构图 PNG URL。
- 早期代号在 FDA 查不到、生物药在 PubChem 查不到 → 标 `❓N/A — 未找到（已查 …）`，不算异常。

### 步骤 2 · 厂家确认（区分三种角色）
明确区分 **原研 / 当前权利人(license holder) / 上市许可持有人**（对用户用"在当地负责上市、卖这药的公司"表述）。按序溯源：
1. **公司官方公告 / 监管批件**（最高 → 🏛️官方 附 URL；对"谁持有批件/谁上市"这类客观事实可标最高可信） →
2. ClinicalTrials.gov / CDE 试验 `sponsor` 字段（✅；脚本 `leadSponsor`、`openfda.drugsfda.sponsor_name`） →
3. 大型机构新闻稿（**标 ⚠️ 需人工确认**）。
- **分别给出美国、中国各自的权利人 / 上市持有人**。
- 中国侧持有人 → 🔴 / G5（chinadrugtrials/NMPA 核对；即便引用了官网，🔴 仍保留）。

### 步骤 3 · MoA（作用机制）
优先级：
1. **药企官网 / FDA 说明书**（客观机制描述 → 🏛️官方 附 URL） →
2. **ChEMBL `mechanism_of_action`**（数据库级，✅ChEMBL；脚本 `chembl.mechanisms`，含 target_chembl_id） →
3. 临床试验 info 描述（标 NCT 号） →
4. 网络资源（**标 ⚠️**）。

### 步骤 4 · 最高临床阶段 / 时间线（🚧 G2）
依开场 B 的口径：
- **「最高进度」**：直接定位最晚阶段（脚本 `clinicaltrials_gov.overall_highest_phase` + openFDA/NMPA 上市状态）。
- **「按适应症」**：
  1. 先确认有多少适应症在研/已上市（从 `studies[].conditions` 聚类 + web 校正）。
  2. 对每个适应症，抓 **ClinicalTrials.gov（脚本 studies）+ chinadrugtrials.org.cn/CDE** 的相关试验，按 **上市 > III > II > I** 排序。
  3. 若已上市：**批准日期/获批适应症**若来自 FDA 批准页或 NMPA → 🏛️官方；其 **pivotal 试验 NCT/CTR 号 → 🔴 + 🚧 G2**（CT.gov 无 "pivotal" 字段，须从 FDA 说明书 *Clinical Studies* / CDE 上市依据读取，必经人工确认）。
  4. 为每个适应症各阶段编写**时间线**（开始/主要完成/完成 日期；脚本 `startDate / primaryCompletionDate / completionDate`）。
- ⚠️ **试验→适应症映射是自由文本、很脏**；任何节点不确定就标 🔴。
- 中国侧阶段/试验 → 🔴 / G5。

### 步骤 5 · 简介（🚧 G4 销售额）
- 若已上市：摘公司官网主要介绍（产品定位等客观描述 → 🏛️官方；带宣传色彩的措辞 → 🔍web/⚠️）。
- **销售成绩 → 🔴 + 🚧 G4**：销售数字只在公司**财报/年报**。给"数字 + 财报出处(年份/季度/页)"；取不到写 `❓N/A`。
  **即使数字直接摘自公司官方财报，也写成 `🔴（来源🏛️官方财报，财务口径仍须复核）`——官方源不解除此处 🔴。绝不估算/编造数字。**

### 步骤 6 · LOE（专利/独占到期）— 美国 + 中国（整项强制 🔴 + 🚧 G3）
- **美国**：查 **FDA Orange Book**（小分子）/ **Purple Book**（生物药）。无现成 "LOE=某日" → 综合**最晚放开仿制的专利 + 独占期**判断。
  产出："预计 LOE + 依据的专利号/独占类型 + 来源 URL"。**专利号本身**若来自 Orange/Purple Book 或专利局可叠 🏛️官方；但"推算出的到期日"整体仍 🔴。
- **中国**：无 Orange Book 等价干净库 → 中国上市药品专利信息登记平台 / 行业数据库 / 新闻**推算**（🔴 / G5）。
- **整项强制 🔴**：必须列依据，绝不只给一个孤立日期；取不到写 `❓N/A` 并说明为何难取。

### 步骤 7 · 是否已有仿制药 / 生物类似药上市 — 美国 + 中国
- **美国**：openFDA / Orange Book（小分子 ANDA）或 Purple Book（生物类似药）是否有获批 → **是/否 + 首个获批年（若有）**。
  脚本信号：`openfda.drugsfda.has_anda_generic`、`openfda.ndc.marketing_categories.ANDA`（✅openFDA）；Purple Book 无类似药 → 可 🏛️官方 标"暂无"。
- **中国**：NMPA 是否有**同通用名仿制批文 / 通过一致性评价 / 生物类似药**记录 → **是/否**（🔴 / G5）。
- 输出**布尔结论 + 来源**；查不到写 `❓N/A`。

### 步骤 8 · 同类药品
分两组各列几个名字（**不展开**）：
- **同 MoA/靶点** 的药（用 ChEMBL 靶点 `target_chembl_id` / 已知药物类别，标可信度）；
- **同适应症** 竞品（从 CT.gov `conditions` / web）。每组标可信度徽标。

### 步骤 9 · 核心优势（AI 总结 · 整节 🔴）
对比 **同类型药品** 与 **同适应症药品** 各自的核心优势：
- **最主观高幻觉区**：每条带来源（🔍web / ✅ / 引用官网时可注明🏛️出处）。
- 加免责：**"跨试验比较在统计上不严谨，仅供方向性参考"**。
- **整节标 🔴 需专家复核**（即便引用官网原文，🔴 也不解除——官网是宣传立场）。

## 7. 输出模式分叉（默认文字 / 按需可视化）

> **默认 = chat 文字报告**（见 7.A，保持现状）。
> **当用户要"文档 / HTML / PDF / 可视化 / 报告文件 / 能存档 / 能分享的版本"时 → 走可视化输出（见 7.B）。**
> 两种模式**内容口径完全一致**：徽标照带、🔴 照旧醒目、末尾核对清单照旧。可视化只是把同样严谨的内容画好看。

### 7.A 文字报告模板（默认）

```markdown
# 药物情报报告：<INN / 商品名>（美国 + 中国）
> 仅供研发情报参考，非医疗/投资建议。生成时间：<UTC>。口径：<最高进度 / 按适应症>。
> 徽标：🏛️官方 最高可信(仅事实) ｜ ✅库 可核实 ｜ 🔍web 附URL ｜ ⚠️ 存疑 ｜ 🔴 高幻觉/必核 ｜ ❓N/A 查过未找到

## 0. 身份卡   （INN/中文名/代号/商品名/CID/ChEMBL ID/盐型/分子式/结构图 …各带徽标）
## 1. 分子名确认
## 2. 厂家（🇺🇸 原研/权利人/上市持有人 ｜ 🇨🇳 原研/权利人/上市持有人）
## 3. MoA
## 4. 最高临床阶段 / 时间线（按口径；pivotal NCT/CTR 🔴）
## 5. 简介（含销售成绩 🔴）
## 6. LOE（🇺🇸 / 🇨🇳，整项 🔴：日期 + 依据 + 来源）
## 7. 仿制药/生物类似药上市？（🇺🇸 是/否+年 ｜ 🇨🇳 是/否 🔴）
## 8. 同类药品（同MoA / 同适应症）
## 9. 核心优势（AI 总结 🔴 + 跨试验不严谨免责）
## 🔴 需人工核对清单   ← 本工具最关键产出
```

### 7.B 可视化报告（HTML / PDF / Word）

触发可视化后，**先照常跑脚本 + web + 各暂停门**拿到全部内容，再用生成器出图。**不牺牲严谨性**：
官方源 🏛️、🔴、核对清单一个不少；好看是加分，**诚实标注可信度仍是第一优先级**。

**做法**：把全部结论填进一份结构化 JSON（schema 见 `templates/report_schema.json`），然后按需选格式：

```bash
# HTML（自包含单文件，可离线打开；浏览器里"打印 → 存为 PDF"即得 PDF）
python scripts/build_report.py <data.json> -o <output.html>
# Word 文档（同一份 JSON → 可编辑 .docx；需 python-docx）
python scripts/build_report_docx.py <data.json> -o <output.docx>
```

> 三种格式（**HTML / PDF / Word**）内容口径完全一致，都是**完整文档**（不是图片截图）。
> 需要 PDF：优先用浏览器对 HTML "打印 → 存为 PDF"，或 `weasyprint <html> <pdf>`，或用 `pdf` skill；
> 也可对 .docx 用 LibreOffice 转 PDF（见 `docx` skill）。

生成的报告**至少含以下组件**，徽标在可视化里用**不同颜色 chip** 区分
（🏛️官方=蓝、✅库=绿、🔍web=青、⚠️=琥珀、🔴=红、❓N/A=灰）：

1. **顶部身份卡 + 免责条**：含徽标彩色 chip。
2. **研发 / 审批时间线**：用**清爽的竖向时间轴**——美国在左(蓝)、中国在右(红)分色，卡片留足空间、
   **绝不因标签过长而被裁切**；每个里程碑点带徽标色。（不要用会溢出裁切的横向布局。）
3. **适应症矩阵**：行=适应症，列=🇺🇸/🇨🇳，格子=状态 pill + pivotal 试验 + 徽标。
4. **销售趋势**：柱状图；预测值用斜纹，每个数据点带 🔴。
5. **🔴 需人工核对清单**：红色高亮区块，醒目保留——最关键产出，**不能因变好看而弱化**。
6. **参考来源（Reference list）**：把全文**所有 source 链接统一汇总到最后一个 section**，
   编号 + 可点击链接 + 徽标；正文里不必散落长 URL，引用都收口到这里。

> 生成的文件放到输出目录，用 `present_files` 给用户（附一句简短说明，不长篇赘述）。
> 配色 / 版式参考 `templates/report_template.html`；要更精细可结合 `frontend-design` 等 skill。

## 8. 末尾「🔴 需人工核对清单」生成规则（最关键产出 · 两种模式都必出）

报告末尾**必须**生成一节 `🔴 需人工核对清单`，把全篇**所有 🔴 / ⚠️ / ❓N/A 汇总成 checklist**。每条含：

```markdown
| # | 字段 | 当前值/状态 | 为什么要核 | 建议去哪个官方源核 |
|---|------|------------|-----------|------------------|
| 1 | 步4 pivotal 试验 NCT | <候选或❓N/A> | CT.gov 无pivotal字段，映射脏 | FDA 说明书 Clinical Studies / CDE 上市依据 |
| 2 | 步5 年销售额 | <数字或❓N/A> | 财报级、易幻觉（官方源也不解除）| 公司年报/季报 |
| 3 | 步6 美国 LOE | <日期+依据或❓N/A> | 需综合专利+独占期推算 | FDA Orange/Purple Book |
| 4 | 步6 中国 LOE | … | 无干净库 | 中国专利登记平台/行业库 |
| 5 | 步7 中国仿制/类似药 | … | 无干净API | NMPA / CDE |
| 6 | 步2/4 中国上市持有人/试验 | … | chinadrugtrials动态、无API | chinadrugtrials.org.cn / NMPA |
| 7 | 步9 核心优势 | AI主观 | 跨试验比较不严谨（官网是宣传）| 领域专家复核 |
| … | 任何 ❓N/A | 查过未找到 | 显式列出 | 列出已查的源 |
```

清单要把**所有**带 🔴/⚠️/❓N/A 的字段都收进来（含中国侧每一项、含"虽来自官方但仍强制 🔴"的销售额与核心优势），不得遗漏。

## 9. 自动化可靠性（执行时心里有数）

- **稳（脚本 ✅，可靠自动化）**：分子身份/结构（PubChem）、ChEMBL ID 与 MoA、CT.gov 阶段/计数/时间线、美国 openFDA 批准与 ANDA 信号。
- **最高可信但仍需附 URL（🏛️官方）**：FDA 批准页/说明书、Purple/Orange Book、公司官网与监管站上的**客观事实**（批准日期、获批适应症、机制、批件状态、专利号）。
- **每次都靠 web、可能不稳（🔍web/⚠️）**：权威医药媒体、二手聚合站。
- **强制 🔴（不因来源权威而解除）**：销售额/财务、核心优势/主观对比、中美 LOE 推算、**整个中国侧**。
- 失败处理：脚本任一源 `null` → 当作"该源没查到"，继续其它源与 web；**绝不**因一源失败（或环境域名未放开）就下"药不存在"。

> 再次强调本工具的价值：不在"给一个漂亮答案"，而在**诚实地标注每个字段的可信度**，
> 并把不确定的东西清清楚楚交到人手上核对。宁可 `❓N/A` 也不编。
