# 安装与上手指南（新手版 · 看这一篇就够）

> 这篇是写给**第一次用、或者不太懂技术**的朋友的。
> 你不需要懂什么"代码""接口""数据库"——照下面一步步点就行。读完大概 5 分钟，跑通一次大概 1 分钟。

这个工具叫 **drug-intel（药物情报）**。你对它说一句「**查一下 XX 药**」，
它就帮你把这只药在 **美国 + 中国** 两地的情况整理成一份报告：临床做到哪一步了、是哪家公司的、
怎么起作用、专利大概什么时候到期、有没有仿制药、同类药有哪些、各自强在哪。

它最大的特点是**诚实**：报告里每一条都标了"这条信息有多可信"，最后还专门给你一张
**"🔴 需人工核对清单"**，把"最容易出错、最好你自己再看一眼"的地方挑出来，告诉你该去哪个官网核。

> ⚠️ 一句话免责：这是**研发情报参考，不是医疗建议、也不是投资建议**。
> 报告里凡是标了 🔴 的地方，请务必自己再核对一遍再用。

---

## 第 1 步：打开"联网搜索"，并把网站"放行"（最关键，先做这个）

这个工具要去**真实的官方网站**上查最新信息（比如美国 FDA、中国药监局、药企官网）。
所以有两个开关要先确认：

### 1A. 打开"联网搜索"

"联网搜索"就是允许 Claude 上网查资料。打开方法：

1. 打开 Claude 桌面版（或网页版）。
2. 进 **设置（Settings）→ 功能 / Capabilities**（不同版本可能叫"工具""Connectors"，找带"网络/搜索/Web"字样的那一项）。
3. 把 **联网搜索 / Web search** 打开（开关变成"开"的状态）。

> 没有这个开关、或者你是公司统一管理的账号？那可能由管理员控制，找你们的 IT/管理员开一下就行。

### 1B. 把它要访问的网站"放行"（这一步最容易卡住，请重点看）

有些环境为了安全，会限制 Claude 只能访问"白名单"里的网站。**如果网站没被放行，工具就会查不到、抓取失败、报告一片空白。**
你有两个选择，**任选其一**：

**选择 A（最省事，推荐新手）：把允许访问的网站设成"全部"。**
在网络/域名设置里，把"允许的域名"填一个星号：

```
*
```

一个 `*` 就代表"所有网站都允许"。设完保存即可，不用记下面那一长串。

**选择 B（不想全开，就只放行下面这些）：把这份清单逐条加进白名单。**
这些就是这个工具会用到的全部网站，复制粘贴进去就行：

```
# —— 工具自动抓数据要用的（必须放行，否则脚本会全部失败）——
pubchem.ncbi.nlm.nih.gov        # 美国 NIH 化合物库（分子身份、结构）
www.ebi.ac.uk                   # 欧洲 ChEMBL 库（靶点、作用机制）
clinicaltrials.gov              # 美国临床试验登记库
classic.clinicaltrials.gov      # 上面这个的旧版地址
api.fda.gov                     # 美国 FDA 开放数据（批准、厂家、仿制药信号）

# —— 美国官方 / 监管网站（查上市、专利、生物类似药）——
www.fda.gov                     # 美国 FDA 官网
www.accessdata.fda.gov          # FDA 橙皮书（小分子专利/独占）
purplebooksearch.fda.gov        # FDA 紫皮书（生物药/生物类似药）

# —— 中国官方网站（查中国的审批、上市、专利、仿制）——
www.chinadrugtrials.org.cn      # 药物临床试验登记与信息公示平台（CDE 官方）
www.nmpa.gov.cn                 # 国家药品监督管理局（NMPA）
www.cde.org.cn                  # 药品审评中心（CDE）
zldj.cnipa.gov.cn               # 中国上市药品专利信息登记平台

# —— 药企官网（看具体查哪只药；下面是常见的几个，按需补充）——
astrazeneca.com                 # 阿斯利康（全球）
astrazeneca.com.cn              # 阿斯利康中国
astrazeneca-us.com              # 阿斯利康美国
daiichisankyo.com               # 第一三共

# —— 可选：跨地区核对 + 文献（不放也能跑，放了更全）——
ema.europa.eu                   # 欧洲药监局（交叉核对用）
ncbi.nlm.nih.gov                # PubMed 等文献
```

> 💡 小贴士：药企官网那几行，是**根据你要查哪只药来定的**。比如查"Enhertu"会用到阿斯利康和第一三共；
> 查别的药可能用到别的公司官网。**懒得逐个补？那就直接用上面的选择 A（填 `*`），一劳永逸。**

设置在哪里？一般在 **设置 → 功能 / Capabilities → 网络访问 / Network**。
如果你是**团队/企业版**，这个开关可能由你们组织的"所有者（Owner）"在管理后台里控制，找管理员放行即可。

## 第 2 步：发起一次查询（就打一句话）

放行好网站后，回到聊天框，**直接用大白话打一句就行**，比如下面任意一种：

> 查一下 Enhertu
>
> 帮我做一下 trastuzumab deruxtecan 的竞品分析
>
> 优赫得（德曲妥珠单抗）的临床进展和专利到期帮我看看

就这么简单。药名可以是**中文名、英文名、商品名、研究代号**里的任意一种，它都能认。

> 想要一份**能存档、能发给同事的好看文档**？在上面那句话后面加一句就行：
> 「**给我出一份 HTML / PDF 的可视化报告**」。它会画出带时间线、适应症对照表、销售趋势图的版本。

---

## 第 3 步：它会先问你一两个问题（别慌，这是正常的）

你发出请求后，它**不会马上闷头写一大篇**，而是先很客气地问你一两个小问题，确认方向。大概是这样：

1. **"你要查的是不是这只药？"** —— 它会把名字、别名列出来让你确认一下，别认错药。
   （如果一个名字能对到好几只药，它会列出来让你选一个。）
2. **"你想看最高进度，还是逐个适应症？"** —— "最高进度"是一句话告诉你这药最快做到了哪一步；
   "逐个适应症"会按乳腺癌、胃癌、肺癌……分别给你，更细更长。
3. **"要不要一次写完？"** —— 你可以让它"每遇到不确定的地方就停下来问你一句"，
   也可以让它"一口气先写完整初稿、把要核对的地方全用红色标在最后"。
   **不确定就选后者**，省事，最后照着红色清单核就行。

你简单回一句（比如"对，就这只""按适应症""一次写完"），它就开始干活了，然后给你报告。**全程你只需要回答这一两句。**

---

## 第 4 步：看懂报告里的"小徽章"（一眼判断哪条能信）

报告里每条信息后面都有一个小标记，告诉你**这条有多可信**。记住这 6 个就行：

| 标记 | 一句话意思 |
|------|-----------|
| 🏛️ 官方 | **最可信**：来自政府/监管官网（FDA、药监局）或药企官网的**客观事实**（比如批准日期、获批适应症）。|
| ✅ | 也很可信：能在权威**数据库**里查到（机器可核实）。|
| 🔍 web | 网上查到的，**会附上链接**，你可以点进去自己看。|
| ⚠️ | 来源有点弱（比如只有一篇报道），**建议你再确认一下**。|
| 🔴 | **最需要你自己再核一遍**的地方（比如销售额、专利到期、中国的审批）。绝不会没来源就瞎给结论。|
| ❓ N/A | 认真查过了，但**确实没找到**（会告诉你查了哪些地方）。|

> 一个重要细节：就算某条信息来自**官网**，只要它是**销售额数字**或**"哪个药更好"这种主观比较**，
> 它**仍然会标 🔴**。因为财报数字容易看错、容易过期，而官网夸自己的话天然带立场。
> 这正是这个工具靠谱的地方——**官方来源能让"事实"更可信，但不会替"宣传/主观"背书。**

报告**最后那张「🔴 需人工核对清单」是重点**：它把全篇所有红色、问号的地方汇总成一张表，
逐条告诉你"为什么要核、去哪个官网核"。你忙的话，至少把这张表过一遍。

---

## 第 5 步：万一"搜不到 / 抓取失败 / 报告一片空白"——八成是网站没放行

如果你发现报告里大片都是"❓没找到""抓取失败""脚本报错"，**最大概率不是这只药有问题，而是第 1 步的网站没放行。**
照下面修一下，基本都能解决：

1. 回到 **设置 → 功能 / Capabilities → 网络访问**。
2. 把"允许的域名"直接改成一个星号 `*`（最省事）；
   或者对照**第 1B 的清单**，看看是不是漏加了某个网站（尤其是 `clinicaltrials.gov`、`api.fda.gov`、`pubchem.ncbi.nlm.nih.gov` 这几个）。
3. 确认**联网搜索**开关是开着的。
4. 保存，回到聊天框，**把刚才那句"查一下 XX 药"再发一次**。

> 还是不行？如果你是**公司/团队账号**，网络白名单可能被管理员锁住了——把第 1B 的清单发给你们 IT/管理员放行即可。
> 记住一条：**某个网站没查到 ≠ 这只药不存在**。早期研究代号、生物药在某些库里查不到，都是正常的。

---

## 第 6 步：怎么把这个工具装上 / 更新（如果还没装）

如果你已经能用了，跳过这步。如果是第一次装、或者要更新到新版：

1. 在这个项目文件夹里，让 Claude（或你自己）跑一下打包脚本，会生成一个 `drug-intel.zip`：
   ```bash
   python build_skill.py
   ```
2. 打开 **Claude 桌面版 → 设置 / Customize → Skills（技能）→ Upload skill（上传技能）**，选这个 `drug-intel.zip`。
   （如果上传框只认 `.skill` 后缀，把文件名改成 `drug-intel.skill` 就行，内容完全一样。）
3. 装好后，回聊天框打「查一下 XX 药」就能用了。

## 最后再叮嘱一句

这个工具是帮你**快速把一只药的全貌摸清楚、并诚实告诉你哪些地方别全信**。
它**不替你做医疗或投资决定**。报告里所有 🔴 的地方，请你（或专业同事）务必到官网再核一遍再用。
